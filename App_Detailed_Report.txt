OmniFlix App Detailed Report
==========================

1. Application Overview
-----------------------
The application is a full-stack MERN (MongoDB, Express, React, Node.js) styled streaming platform named "OmniFlix". It is designed to resemble Netflix while keeping an underlying "Ben 10 theme". The app supports advanced features such as user authentication, profile creation (with alien avatars), browsing movies using external APIs (TMDB and OMDb), an advanced search function, and my-list synchronization.

2. Architecture
---------------
The project contains two main parts under a monorepo-like structure in the current root folder:
- **Client**: The frontend user interface constructed with React and Vite.
- **Server**: The backend RESTful API built with Express, Node.js, and MongoDB.

3. Client-Side (Frontend)
-------------------------
The standalone client lies in the `client/` folder, engineered with Vite, React 19, and TypeScript.
• Core Libraries:
  - Framework: React (v19.x), react-router-dom for navigation.
  - State Management: Zustand.
  - Animations: framer-motion.
  - Styling: Tailwind CSS (v4) with vanilla CSS (`index.css`).
  - API Communications: Axios.
  - Media & Utility: react-player, react-icons, react-intersection-observer.

• Directory Structure (`client/src/`):
  - **components/**: Segregates reusable pieces corresponding to distinct areas like layout, movie displays, and ui components.
  - **pages/**: Contains routing destinations encapsulating full views, like AccountPage, BrowsePage, LandingPage, LoginPage, MyListPage, ProfileSelectPage, and SearchPage.
  - **store/**: Likely houses Zustand logic.
  - **hooks/**, **services/** (API connectors), **types/**, and **assets/** / **data/** compose the rest of the application.

4. Server-Side (Backend)
------------------------
The core backend lies under the `server/` folder and serves data securely to the frontend using REST APIs.
• Core Libraries:
  - Framework: Express.js inside Node.
  - Database & ORM: MongoDB connected via Mongoose.
  - Validation: Zod.
  - Security & Authentication: bcryptjs, jsonwebtoken (JWT), helmet, express-rate-limit, cors.
  - Performance: node-cache.
  - Logging: winston, morgan.
  - Types definitions for TypeScript support and the `tsx` tool for rapid execution.

• Directory Structure (`server/src/`):
  - **models/**: Houses Mongoose schemas. Notably contains `User.ts`.
  - **routes/**: Divided logically for the endpoints logic. Contains `auth.ts`, `profile.ts` (acting on user endpoints), alongside external interaction proxies `tmdb.ts` and `imdb.ts` pulling streaming metadata.
  - **controllers/**: Handles the business functionality separate from routing.
  - **services/**: Additional services such as tmdbService and imdbService integrations.
  - **middleware/**, **config/**, **types/**, and **utils/** ensure application stability.

5. Key Takeaways
----------------
- Highly modularized architecture on both sides enforcing a strict separation of concerns globally.
- Clean integration with third-party service endpoints (TMDB, OMDb/IMDb).
- Substantial UI components built indicating a robust, styled presentation layer featuring dynamic animations via framer-motion and modern component building (React 19, Tailwind CSS).
- Secure and validated backings using standard best practices involving token management (JWT), payload verifications (Zod), and secure database storing (Mongoose).
